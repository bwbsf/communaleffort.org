# Research Integration Validation Governance

Key: `[ ]` pending task, `[x]` completed task, `[?]` needs validation, `[-]` closed task

## Goal

Update research governance so opportunity integration is not trusted merely because a completed report exists. Every proposed opportunity must pass explicit validation checks for existence, accuracy, and collaboration fit before it enters a chapter page.

## Checklist

- [x] Update the integration playbook to cover regular-prompting reports as well as deep-research reports.
- [x] Add required source validation checks for organization existence, active status, local relevance, contact pathways, claim support, category fit, and collaboration appropriateness.
- [x] Require failed validation to block integration and produce a user-facing explanation of the failed checks.
- [x] Update research governance docs so completed reports remain preserved but are treated as untrusted inputs until validated.
- [x] Update agent-facing standards to require the validation gate before importing research findings.
- [x] Add a journal entry for the governance change.
- [x] Regenerate plan indexes after creating and updating this plan.
- [x] Verify the changed docs mention the validation gate and failure behavior.
